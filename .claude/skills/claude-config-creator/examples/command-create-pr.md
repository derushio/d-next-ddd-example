# Create PR Description

## 目的

現在のブランチの変更内容を分析し、プルリクエスト用の説明文を生成する。

## 実行手順

1. 現在のブランチ名を取得
2. mainブランチとの差分を取得（`git diff main...HEAD`）
3. コミット履歴を取得（`git log main..HEAD --oneline`）
4. 変更内容を分析して以下の構成で説明文を生成:
   - タイトル（50文字以内）
   - 概要（変更の目的）
   - 変更内容（箇条書き）
   - テスト方法
   - 関連Issue（あれば）

## 入力

- **必須**: なし（現在のブランチを使用）
- **オプション**: 関連するIssue番号

## 出力

```markdown
## タイトル
{簡潔なタイトル}

## 概要
{この変更が何を解決するか}

## 変更内容
- {変更1}
- {変更2}

## テスト方法
{テスト手順}

## 関連Issue
- #{issue番号}
```

## 使用例

```
/create-pr
/create-pr #123
```

## 注意事項

- mainブランチが最新であることを確認
- コミットメッセージが適切に書かれていると精度が上がる
